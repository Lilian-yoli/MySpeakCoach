# Gemini Live English Role-Play - Design Specification

## Overview
A web-based intelligent coaching application built with React and Vite. It provides an immersive, voice-first role-play experience using real-time AI to help users practice their spoken English. The focus is on uninterrupted conversational flow, followed by actionable post-session feedback.

## User Experience Flow
1. **Initiation:** User launches the app and clicks a single "Start Session" button.
2. **Immersive UI:** The interface shifts to a minimal, distraction-free visual state (e.g., a pulsing orb that reacts to voice activity).
3. **Voice-First Setup:** The AI greets the user with voice: *"Hello! What topic or scenario would you like to role-play today?"*
4. **Role-Play:** The user speaks their desired scenario (e.g., "A job interview for a software engineer"). The AI immediately adopts that persona and begins a continuous, natural back-and-forth audio conversation.
5. **Session End:** The user clicks an "End Session" button to terminate the active role-play.
6. **Post-Session Refinement:** The immersive view closes and is replaced by a dashboard. It displays the user's side of the transcript alongside "Native Speaker Refinements"—polished versions of their sentences generated by the AI to improve vocabulary, grammar, and natural tone.

## Core Architecture
* **Frontend Framework:** React (Vite) Single Page Application.
* **Audio Handling:** `MediaRecorder` API for capturing microphone input. Web Audio API for audio playback.
* **Real-time Engine:** WebSockets connecting to the AI backend (e.g., Gemini Multimodal Live API or an orchestrated STT -> LLM -> TTS pipeline proxy) to stream audio data bidirectionally for ultra-low latency.
* **State Management:**
  - `SessionManager`: Handles the WebSocket connection lifecycle and streaming state.
  - `TranscriptStore`: Silently logs the conversation transcript during the free-flowing role-play.
* **Post-Processing:** After the real-time session terminates, a standard LLM invocation evaluates the saved `TranscriptStore` to generate the refinement feedback.

## Key Components
1. `App`: Orchestrates routing between the Landing Page, the Active Session view, and the Results Dashboard.
2. `ImmersiveOrb`: A highly visual component that reacts (pulses/scales) based on current audio metrics (user speaking vs AI speaking). 
3. `RefinementDashboard`: A card-based UI that iterates over user utterances, displaying the original speech next to the AI's "Native Tone" suggestions.
4. `AudioStreamer`: A utility hook to manage the lifecycle of microphone access and audio buffer streaming.

## Error Handling & Edge Cases
* **Microphone Permissions:** Clean error UI if `getUserMedia` access is denied.
* **Connection Drops:** Auto-reconnect logic for WebSocket failures with a non-intrusive visual indicator.
* **Silence Handling:** Timeout mechanics if the user doesn't speak during the initial setup or mid-conversation for an extended period, prompting the AI to politely check in.
